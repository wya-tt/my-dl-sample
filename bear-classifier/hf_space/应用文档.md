## 一、目的

这是一个基于深度学习的**图像分类 Web 应用**，用于识别上传的图片属于以下三类熊中的哪一种：

| 类别 | 含义 |
|------|------|
| `black`   | 黑熊 |
| `grizzly` | 灰熊（棕熊） |
| `teddy`   | 泰迪熊（玩具熊） |

模型来自 fastai 课程《Practical Deep Learning for Coders》的经典示例，训练数据为通过图片爬虫收集的三类熊图片，使用迁移学习（ResNet）微调得到。本应用的目标是把训练好的模型**部署成一个任何人都能在浏览器里直接使用的在线 demo**。

## 二、使用方法

1. 打开应用链接。
2. 在左侧 **"Upload a bear image"** 区域上传一张图片（点击上传或拖拽，支持常见图片格式）。
   - 页面底部提供了 3 张示例图（黑熊 / 灰熊 / 泰迪熊），可直接点击试用。
3. 模型会自动进行预测，右侧 **"Prediction"** 区域显示结果：
   - 给出最可能的类别及各类别的**置信度概率**（最多显示 3 类）。
4. 想换图重测，重新上传即可。

> 首次访问若 Space 处于休眠状态，会有几十秒的唤醒/加载时间，属正常现象。

## 三、主要设计思路

### 1. 训练与推理分离
- **训练**在本地完成（见 `02_production.ipynb`），训练好的模型通过 `learn.export()` 导出为单个文件 `export.pkl`（约 45 MB），其中包含网络结构、权重、数据预处理流程（item/batch transforms）与类别词表 `vocab`。
- **推理/部署**只需加载 `export.pkl`，无需训练代码与原始数据，应用启动即 `load_learner("export.pkl")`。

### 2. 用 Gradio 构建交互界面
- 采用 `gr.Interface` 这一高层封装：输入为图片、输出为带概率的标签，几行代码即可生成完整网页 UI，无需手写前端。
- 输出用 `gr.Label(num_top_classes=3)`，直观展示各类别置信度。

### 3. 选择 Hugging Face Spaces 而非 Binder 部署
- 最初尝试用 Binder + Voilà 部署，但 Binder 会话**空闲即回收**、且运行子域名网络可达性差，多次出现 `Binder inaccessible` / `424` 等问题，不适合长期稳定访问。
- 改用 **Hugging Face Spaces**：免费、持久、单域名访问稳定，闲置只是休眠、访问即唤醒，更适合做长期可用的 demo。

### 4. 解决跨平台 / 跨版本的部署坑（关键工程细节）
模型在 **Windows + Python 3.13** 上导出，部署到 **Linux** 的 Space，踩了一系列坑并逐一解决：

- **`WindowsPath` 无法在 Linux 重建**：在 `app.py` 加载模型前，将 `WindowsPath` 映射为 `PosixPath`（仅在 POSIX 系统上执行）。
- **`pathlib._local` 模块**：Python 3.13 才有，因此 Space 也固定使用 **Python 3.13**（与导出环境一致）。
- **`pydub` 依赖 `audioop`**：Python 3.13 移除了标准库 `audioop`，通过加入 backport 包 **`audioop-lts`** 解决。
- **pickle 反序列化所需的包**：模型内部引用了 `cloudpickle`、`fasttransform`、`plum`(plum-dispatch)、`numpy(_core)` 等，在 `requirements.txt` 中固定对应版本，保证与导出环境一致，避免反序列化失败。
- **依赖版本一致性**：`fastai`、`fastcore`、`torch`、`numpy` 等钉到与训练导出时相同的版本，确保模型能被正确加载。
- **Gradio 版本策略**：直接使用**最新版 gradio（6.x）**，由它自动拉取配套的 `fastapi`/`starlette`/`pydantic`/`huggingface_hub`，避免"旧版 gradio 撞上新版传递依赖"导致的接口不兼容问题。

## 四、用到的主要模块

### 部署项目文件结构
```text
hf_space/
├── app.py              # Gradio 应用主程序
├── requirements.txt    # 依赖声明
├── README.md           # HF Space 配置（sdk、python 版本等）
├── export.pkl          # 训练导出的 fastai 模型（约 45MB）
└── examples/           # 示例图片
    ├── black.jpg
    ├── grizzly.jpg
    └── teddy.jpg
```

### 核心 Python 模块 / 库

| 模块 / 库 | 作用 |
|-----------|------|
| **fastai** (`fastai.vision.all`) | 加载模型 `load_learner`、图像封装 `PILImage`、执行预测 `learn.predict` |
| **gradio** | 构建 Web 交互界面（`gr.Interface` / `gr.Image` / `gr.Label`）并启动服务 `demo.launch()` |
| **PyTorch** (`torch` / `torchvision`) | fastai 的底层深度学习框架，承载 ResNet 模型的张量运算与推理 |
| **fastcore / fasttransform / plum-dispatch** | fastai 的底层依赖，模型反序列化时被引用 |
| **cloudpickle** | 模型 pickle 中部分对象的序列化/反序列化支持 |
| **numpy** | 数值数组运算（模型 pickle 引用 `numpy._core`） |
| **PIL (Pillow)** | 图像读取与处理 |
| **pathlib** | 路径处理；用于跨平台 `WindowsPath → PosixPath` 兼容补丁 |
| **audioop-lts** | 为 Python 3.13 提供 `audioop`，满足 gradio 依赖的 `pydub` |

### 应用核心逻辑（`app.py`）

```python
import os
import pathlib

# 模型在 Windows(Python 3.13) 导出，Linux 上需把 WindowsPath 映射为 PosixPath
if os.name == "posix":
    pathlib.WindowsPath = pathlib.PosixPath
    if hasattr(pathlib, "_local"):
        pathlib._local.WindowsPath = pathlib._local.PosixPath

from fastai.vision.all import load_learner, PILImage
import gradio as gr

learn = load_learner("export.pkl")
labels = learn.dls.vocab

def predict(img):
    img = PILImage.create(img)
    pred, pred_idx, probs = learn.predict(img)
    return {str(labels[i]): float(probs[i]) for i in range(len(labels))}

demo = gr.Interface(
    fn=predict,
    inputs=gr.Image(type="pil", label="Upload a bear image"),
    outputs=gr.Label(num_top_classes=3, label="Prediction"),
    title="Bear Classifier",
    description="Upload a bear image to predict whether it is a black bear, grizzly bear, or teddy bear.",
    examples=["examples/black.jpg", "examples/grizzly.jpg", "examples/teddy.jpg"],
)

if __name__ == "__main__":
    demo.launch()
```
